# Ian 小工工程师正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做商业插画、PPT 信息图或可爱卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、有技术气质、可读但不说明书的结构化解释图。

默认视觉 IP 是"小工"：Q版电子/软件工程师，约 3~3.5 头身，深棕色短侧分 quiff 发型，黑色中框方眼镜（最核心识别符），暖桃肤色。两套服装：日常深灰蓝衬衫（办公/软件），浅天蓝连体静电服（工厂/硬件/现场）。小工必须参与画面的核心动作，不能只是站在旁边当装饰。

## 场景切换原则

根据文章主题自动选择服装体系：

- 软件编程、架构设计、Bug调试、代码部署、白板讲解 → **体系A：深灰蓝衬衫**
- 硬件调试、工厂产线、户外巡检、设备检修、焊接、配电箱、现场交付 → **体系B：浅天蓝静电服**

不确定时默认使用体系A。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/style-dna.md`：风格 DNA、颜色（含色值）、禁忌。
- `references/xiaohei-ip.md`：小工 IP 的形象、性格、两套服装、动作库和禁止项。
- `references/composition-patterns.md`：结构类型、原创隐喻方法和反复刻规则。
- `references/prompt-template.md`：单张生图提示词模板（含正向/负向关键词）。
- `references/qa-checklist.md`：生成后检查和迭代规则。
- `assets/examples/`：只作低频视觉校准，不进入默认生成路径。不要照抄这些案例的构图。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择"认知锚点"：核心判断、两个断点、输入输出闭环、分流、前后对比、一鱼多吃、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说"分析怎么配图 / 思考哪些地方需要配图"，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 小工在图里做什么
- 服装体系（A 或 B）
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。

### 3. 单张生成

**🚨 核心策略：图生图优先。** 必须用参考底图 `assets/reference/xiaogong-front.png` 作为输入，保持形象一致。

如果用户明确要求"生成 / 输出 / 做图 / 帮我生成"，用内置 `image_gen` 每张单独生成：

```bash
python scripts/generate_image.py \
  --prompt "Same exact character — preserve every detail. Only change pose and scene: {场景描述}" \
  --filename "output.jpg" \
  -i "assets/reference/xiaogong-front.png" \
  --aspect-ratio 16:9
```

**场景 prompt 须包含：**
- 保留所有原始特征（脸、眼镜框厚、深棕侧分 quiff、暖桃肤色、大圆眼、小耳朵、深灰蓝衬衫、数字 cel-shading、3.5 头身）
- 纯白背景
- 小工角色尺寸约束：不超过画面高度的 30%、宽度的 20%。结构图/流程图是画面主体，小工站在旁边讲解
- 小工作为核心动作主体（讲解、指向、连接），但不压倒结构信息
- 必要时切换服装体系 B（浅天蓝静电服）

**仅在没有参考底图时**降级为文生图，使用 prompt-template.md 中的正向+锁定+负面关键词。

不要复刻过往案例。每次都要从当前文章重新发明一个清晰但新颖的视觉表达。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。出现以下问题优先重生成：

- 小工只是装饰
- 小工缺少眼镜，或眼镜是细框/粗框（应该是中框）
- 头发是纯黑（应该是深棕 #3B332D）
- 眼睛是杏仁眼或夸张圆眼
- 画风有蜡笔/手绘纹理（应该是数字 cel-shading）
- 肤色是冷白（应该是暖桃色 #FFF0D5）
- 画面太满
- 太像流程图/PPT
- 左上角出现类型标题
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 服装体系
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
